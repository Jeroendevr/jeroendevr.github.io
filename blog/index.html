Installing development version of Spyder (Version 4.)

written by Jeroen on 2019-10-23

As I have been using SPYDER as one of my main IDE for programming in Python, I wanted to install the beta version however I have challenged myself to download and install this though PIP instead of conda. Official documentation on installing development builds.

Prerequisite

  • Python installation
  • VirtualEnv package
  • GIT

Launch from console

To launch Spyder you have to cd to you SPYDER installation and python bootstrap.py It would be better to launch a application u use often directly from terminal. This can be done by defining a function in your terminal preference file. In my case my shell is zsh as it is default since macOS Catalina. open ~/.zshrc and define your own function within this preference file.

In my case I have defined a function spyder as follows

spyder() {

    cd /Users/jeroendevries/Programming/env_spyder/spyder

    git pull https://github.com/spyder-ide/spyder.git

    python3 bootstrap.py

Python on a Mac

written by Jeroen on 2019-10-21

macOS Catalina

Python can be installed using different ways on a Mac. A how to install is described on the python website. However I like no nonsens and clean installation. So I use a package manager to install all sorts of tools.

HomeBrew

Install Homebrew a package manager for Mac.

Command Line Tools

Next to home-brew you need the xcode command tools. You can install those typing xcode-select --install in the terminal.

If receiving the following already installed message

xcode-select: error: command line tools are already installed, use "Software Update" to install updates[1] Try softwareupdate --install -a to install all available updates

If you where not able to install Python 3.x try reinstall HomeBrew see this Question
